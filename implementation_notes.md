# Пошаговое внедрение в УТ 11

## 1. Создать расширение

Создайте расширение, например:

- `БригадирыЗаказыНаСборку`

Если у вас уже есть рабочее производственное расширение, можно внести изменения туда же.

## 2. Занять документ

Займите документ `Документ.ЗаказНаСборку`.

Если в вашей базе внутреннее имя документа отличается, используйте фактическое имя метаданных.

## 3. Добавить реквизиты документа

Добавьте в документ два реквизита:

### `ПометкаБригадиров`
- Тип: `Булево`
- Заголовок: `Пометка (Бригадиры)`

### `КомментарийБригадиров`
- Тип: `Строка`
- Длина: `500`
- Заголовок: `Комментарий бригадиров`

## 4. Расширить основную форму заказа

На основной форме документа:

1. Найдите место рядом с существующими флажками или блоком комментария.
2. Добавьте поле формы для реквизита:
   - `Объект.ПометкаБригадиров`
3. Подпись:
   - `Пометка (Бригадиры)`
4. Подключите обработчик:
   - `ПометкаБригадировПриИзменении`

Код обработчика лежит в `order_form_module.bsl`.

## 5. Создать рабочее место бригадиров

Проще всего сделать отдельную общую форму, например:

- `ОбщаяФорма.РабочееМестоБригадировЗаказыНаСборку`

### Реквизиты формы

Добавьте реквизит формы:

- `СписокЗаказов`
- Тип: `ТаблицаЗначений`

У таблицы должны быть колонки:

- `Ссылка`
- `Номер`
- `Дата`
- `КомментарийМенеджера`
- `КомментарийБригадиров`

### Элементы формы

На форме разместите:

- таблицу с именем `СписокЗаказов`
- команду `Обновить`
- команду `ОткрытьЗаказ`
- команду `СохранитьКомментарий`
- команду `СнятьПометку`

Модуль формы возьмите из `brigadiers_workspace_form_module.bsl`.

## 6. Добавить общий модуль

Создайте общий модуль расширения, например:

- `БригадирыЗаказыСервер`

Свойства:

- серверный
- вызов сервера разрешён

Код возьмите из `common_module.bsl`.

Этот модуль делает три вещи:

1. Получает список отмеченных заказов.
2. Сохраняет отдельный комментарий бригадиров.
3. Снимает пометку, когда заказ больше не нужен в списке.

## 7. Подключить команду в интерфейс

Добавьте команду открытия общей формы в подсистему `Производство`.

Подходящее имя команды:

- `Заказы бригадирам`

Если в вашей базе уже есть отдельный блок производственных рабочих мест, лучше положить команду именно туда.

## 8. Что проверить сразу после внедрения

1. Открыть заказ на сборку.
2. Поставить флажок `Пометка (Бригадиры)`.
3. Записать заказ.
4. Открыть рабочее место бригадиров.
5. Убедиться, что заказ появился в списке.
6. Заполнить `КомментарийБригадиров`.
7. Открыть заказ повторно и убедиться, что:
   - стандартный `Комментарий` не затронут
   - комментарий бригадиров сохраняется отдельно
8. Снять пометку в рабочем месте и проверить, что заказ пропадает из списка.

## 9. Дополнительный шаг по автозапуску формы

Это лучше делать после согласования списка.

Варианты:

1. Назначить форму стартовой страницей для нужных пользователей через пользовательские настройки.
2. Либо доработать сценарий запуска и открывать форму программно только для выбранных пользователей.

Для первого внедрения безопаснее начать с обычной команды в разделе `Производство`.
