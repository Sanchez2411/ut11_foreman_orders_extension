# Расширение УТ 11: заказы на сборку для бригадиров

## Что решаем

В `Заказе на сборку (разборку)` менеджер должен поставить отдельную пометку, если заказ нужно показать бригадирам как срочный или требующий отдельного контроля.

После установки пометки заказ должен появиться в отдельном рабочем месте бригадиров. В этом рабочем месте бригадиры видят список отмеченных заказов и могут писать свой собственный комментарий, не трогая стандартный комментарий заказа.

## Согласованный сценарий

1. Менеджер открывает заказ на сборку.
2. Ставит флажок `ПометкаБригадиров`.
3. Заказ попадает в отдельный список бригадиров.
4. Бригадиры работают в своём списке и заполняют `КомментарийБригадиров`.
5. Стандартный реквизит `Комментарий` в заказе остаётся для менеджеров.

## Архитектура решения

### 1. Расширяем документ

Нужно занять документ `Заказ на сборку (разборку)` и добавить в него реквизиты:

- `ПометкаБригадиров` (`Булево`)
- `КомментарийБригадиров` (`Строка`, например 500 символов)

### 2. Дорабатываем форму заказа

На основной форме заказа выводим только флажок `ПометкаБригадиров`.

Отдельный комментарий бригадиров на форму самого заказа можно не выводить, чтобы не смешивать комментарии менеджеров и производства.

### 3. Делаем рабочее место бригадиров

Нужно создать отдельную форму или отдельную команду списка, где показываются только заказы с `ПометкаБригадиров = Истина`.

В список стоит вывести минимум такие поля:

- `Ссылка`
- `Номер`
- `Дата`
- `Комментарий` менеджера
- `КомментарийБригадиров`

### 4. Добавляем точку входа

Команду рабочего места нужно поместить в подсистему `Производство`.

### 5. Дополнительное пожелание

Для пользователей бригадиров можно настроить стартовое открытие этого рабочего места при входе в систему. Это удобнее делать уже после основного внедрения, когда форма будет согласована.

## Какие файлы подготовлены

- `order_form_module.bsl` — модуль формы заказа с обработчиком поля `ПометкаБригадиров`
- `brigadiers_workspace_form_module.bsl` — модуль формы рабочего места бригадиров
- `common_module.bsl` — серверные процедуры получения списка и записи комментария
- `implementation_notes.md` — пошаговое внедрение в конфигураторе

## Важное допущение

В большинстве баз УТ 11 документ внутри метаданных называется `Документ.ЗаказНаСборку`, хотя в интерфейсе он показывается как `Заказ на сборку (разборку)`.

Если в вашей базе внутреннее имя другое, замените его в:

- запросе в `common_module.bsl`
- строке открытия формы в `brigadiers_workspace_form_module.bsl`
- шагах внедрения в `implementation_notes.md`
